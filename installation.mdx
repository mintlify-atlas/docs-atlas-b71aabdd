# Installation Guide

This guide covers everything you need to set up Pokémon Explorer on your local machine for development, customization, or contribution.

<Info>
Pokémon Explorer has **zero dependencies**—no npm packages, no build tools, no configuration files. It's pure HTML, CSS, and JavaScript.
</Info>

Before you begin, ensure you have:

## Installation Steps

### 1. Clone the Repository

<Steps>
  <Step title="Clone via HTTPS">
    ```bash
    git clone https://github.com/mueses1/PokeApi.git
    cd PokeApi
    ```
  </Step>
  
  <Step title="Alternative: Clone via SSH">
    ```bash
    git clone git@github.com:mueses1/PokeApi.git
    cd PokeApi
    ```
  </Step>
  
  <Step title="Alternative: Download ZIP">
    1. Visit [https://github.com/mueses1/PokeApi](https://github.com/mueses1/PokeApi)
    2. Click the green "Code" button
    3. Select "Download ZIP"
    4. Extract the ZIP file to your desired location
  </Step>
</Steps>

### 3. Choose Your Development Method

<Tabs>
  <Tab title="File Protocol">
    **Simplest Method** (with limitations)
    
    ```bash
    # Just open the file
    open index.html          # macOS
    start index.html         # Windows
    xdg-open index.html      # Linux
    ```
    
    <Warning>
    Using `file://` protocol may cause issues with:
    - External resources (fonts, CDN libraries)
    - CORS policies
    - Some browser APIs
    </Warning>
  </Tab>
  
  <Tab title="Python Server">
    **Recommended for Quick Setup**
    
    ```bash
    # Python 3 (recommended)
    python3 -m http.server 8000
    
    # Python 2
    python -m SimpleHTTPServer 8000
    ```
    
    Then visit: [http://localhost:8000](http://localhost:8000)
    
    <Info>
    Python is pre-installed on macOS and most Linux distributions.
    </Info>
  </Tab>
  
  <Tab title="Node.js Server">
    **If You Have Node.js Installed**
    
    ```bash
    # Using npx (no installation needed)
    npx http-server -p 8000
    
    # Or install globally
    npm install -g http-server
    http-server -p 8000
    
    # Or use live-server for auto-reload
    npx live-server --port=8000
    ```
    
    Then visit: [http://localhost:8000](http://localhost:8000)
  </Tab>
  
  <Tab title="VS Code">
    **Best for Active Development**
    
    1. Install [Live Server extension](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer)
    2. Open the project folder in VS Code
    3. Right-click `index.html`
    4. Select "Open with Live Server"
    
    <Tip>
    Live Server automatically reloads the page when you save changes—perfect for development!
    </Tip>
  </Tab>
</Tabs>

## Troubleshooting

<Accordion title="Pokémon images not loading">
  **Cause**: Slow network or PokeAPI rate limiting
  
  **Solution**: 
  - Check browser console for 429 errors
  - Wait a moment and refresh
  - Images use lazy loading, so they load as you scroll
</Accordion>

<Accordion title="Search suggestions not appearing">
  **Cause**: JavaScript error or DOM not ready
  
  **Solution**:
  - Check browser console for errors
  - Ensure `logica.js` is loading after DOM elements
  - Verify the `searchSuggestions` element exists
</Accordion>

<Accordion title="Audio not playing">
  **Cause**: Browser autoplay policies
  
  **Solution**:
  - Click the 🔊 button once to activate audio
  - Check browser console for audio errors
  - Ensure PokeAPI cry URLs are accessible
</Accordion>

<Accordion title="Styles not applying">
  **Cause**: CSS file path or cache issues
  
  **Solution**:
  - Verify `styles.css` is in the same directory as `index.html`
  - Hard refresh: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
  - Check Network tab in DevTools to confirm CSS loaded
</Accordion>

<Accordion title="Cannot start local server">
  **Cause**: Port already in use or command not found
  
  **Solution**:
  ```bash
  # Try a different port
  python3 -m http.server 3000
  
  # Or find what's using port 8000
  # macOS/Linux:
  lsof -i :8000
  
  # Windows:
  netstat -ano | findstr :8000
  ```
</Accordion>

const batchSize = 50;
for (let i = 0; i < needed.length; i += batchSize) {
    const batch = needed.slice(i, i + batchSize);
